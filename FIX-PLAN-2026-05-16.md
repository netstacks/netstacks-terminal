## P1 — SECURITY: Active risk to user data

### P1-1. `local_dir_list` + `local_file_exists` bypass the path validator — **XS**
- `agent/src/api.rs:9440-9478` and `9540-9545`. Every sibling local-fs handler calls `validate_local_path(&req.path)?` first. These two don't.
- **Fix:** add the `validate_local_path` call as the first line of each handler. Two-line fix.
- With bearer token, currently allows arbitrary directory enumeration (`POST /api/local/list-dir {"path":"/etc"}`) and sensitive-file existence oracle.

### P1-2. `validate_local_path` is TOCTOU-vulnerable — callers discard the canonical PathBuf — **S**
- `agent/src/api.rs:9410,9427,9488,9505,9526-9527,9556`. Validator returns canonicalised `PathBuf` (line 9399); every caller throws it away and uses the raw user-supplied string. Symlink swap between validate and write bypasses the deny list.
- **Fix:**
  1. Change `validate_local_path` to return a `Result<PathBuf>` (already does).
  2. Change the function signature so callers can't ignore the return value — bind it to a local: `let safe_path = validate_local_path(&req.path)?;` and use `safe_path` everywhere downstream.
  3. Add a `#[must_use]` attribute on the function so future calls trigger a compiler warning if dropped.

### P1-3. Vault `std::sync::Mutex` poisons permanently brick credential unlock — **S**
- `agent/src/providers/local.rs:54,1148,1159,1175-1187,1869,1931,1977,1987-1997`. Vault unlock counter, master vault, master salt — all sync `Mutex`/`RwLock` with `.lock().unwrap()`. A panic anywhere holding any of these poisons it; subsequent vault operations panic the request thread. Combined with the AUDIT-FIX cooldown logic also living behind these locks, one panic permanently bricks the vault.
- **Fix:** swap `std::sync::Mutex`/`RwLock` for `parking_lot::Mutex`/`RwLock` (no poison). For critical sections that cross `.await` boundaries (verify each one — none should), use `tokio::sync::Mutex`. Same for `agent/src/ai/oauth2.rs:126`.

### P1-4. SSE Python child processes never killed on disconnect — **S**
- `agent/src/scripts.rs:1317-1431`, `agent/src/api.rs:9566-9648`. On SSE disconnect, `tx.send(...).await` returns `Err(_)`; `let _ =` swallows it; the spawned read task continues; `uv`-spawned Python child keeps running. `while True: pass` runs forever per disconnect.
- **Fix:**
  1. Wrap the stdout-read loop in `tokio::select!` against a `CancellationToken`.
  2. Drop the token on `tx.send` error → reader task exits → call `child.kill().await` in cleanup.
  3. Same pattern for the api.rs SSE handler.

### P1-5. Hardcoded `danger_accept_invalid_certs(true)` (17 sites) + silent-on-error fallback — **M**
- `agent/src/api.rs:1443,1494,1706,1748,1790,1832,1874,1916,2060,2157` + `main.rs:146` + `discovery/integration_lookup.rs:54` + `ai/oauth2.rs:100` + `ai/providers.rs:432,1198,1232,1256`.
- Worse: the `.unwrap_or_else(|_| reqwest::Client::new())` fallback in api.rs **flips TLS verification back on silently** when the builder fails — a security setting that flips on the error path.
- **Fix:**
  1. Introduce `pub fn http_client(verify_ssl: bool) -> Result<reqwest::Client, BuildError>` in a new `agent/src/http.rs`. Builder returns `Result`, not a fallback Client.
  2. Add a per-resource `verify_ssl: bool` field to NetBox/LibreNMS/Netdisco source models (already exists on `ApiResource` per quick_actions.rs:117).
  3. For AI providers and OAuth2: add `verify_ssl` to the provider config (already has `base_url` for custom — add a sibling field).
  4. Replace all 17 sites with `http_client(source.verify_ssl)?`.

### P1-6. WebSocket auth uses plain `==` (3 sites) — **XS**
- `agent/src/ws.rs:90,1164,1600`. HTTP middleware uses `subtle::ConstantTimeEq` (the AUDIT-FIX comment is right there). The three WebSocket handlers compare the same token with `==`.
- **Fix:** replace each `==` with the same `subtle::ConstantTimeEq` pattern used in `api.rs:135`.

### P1-7. SFTP download response: panic / header injection via filename — **XS**
- `agent/src/api.rs:3604-3617`. `format!("attachment; filename=\"{}\"", filename).parse().unwrap()`. `filename` is user-controlled SFTP basename. `"`, `\r`, `\n` cause `.parse()` to fail → `.unwrap()` panics the request thread.
- **Fix:** use the `axum::http::header::ContentDisposition` builder, or RFC 6266 `filename*=UTF-8''…` percent-encoded. Drop the `.unwrap()`.

### P1-8. NetBox proxy pagination has no max-page cap — OOM vector — **XS**
- `agent/src/api.rs:2122-2126`. Pagination accumulates pages into `Vec<NetBoxDevice>` with no bound. Misconfigured NetBox returning large `count` + working `next` chain → OOM.
- **Fix:** cap at e.g. `MAX_NETBOX_PAGES = 200` (10k devices). Return 502 with "NetBox returned more pages than allowed" if exceeded.

### P1-9. Reconnect token printed to console — **XS**
- `frontend/src/hooks/useEnterpriseSSH.ts:315` — `console.log('[useEnterpriseSSH] Reconnect token received:', token)` prints the full token.
- `frontend/src/main.tsx:71,89` — sidecar auth token logged.
- `frontend/src/main.tsx:209` — bootstrap error handler `innerHTML = error.message` unescaped (XSS if Tauri produces HTML in an error string).
- **Fix:** remove the token logs entirely. Replace `innerHTML` with `textContent` at `main.tsx:209`.

### P1-10. Vault has no `changeMasterPassword` / `wipeVault` — **L**
- `grep -rn "change_master|changeMaster|wipeVault|resetVault"` returns zero hits. `setMasterPassword` is set-once. A user who wants to rotate their master password has no recourse short of deleting the credential store file.
- **Fix:** add backend endpoints `PUT /api/vault/master-password` (takes old + new, re-encrypts all entries with the new KEK) and `POST /api/vault/wipe` (requires password confirmation, deletes all entries). Wire UI into `VaultSettings.tsx`.

### P1-11. Host keys: no list, no revoke — TOFU mistakes are permanent — **M**
- Backend `agent/src/ssh/host_keys.rs` has only `trust_key` and `verify_or_store`. No `list_host_keys`, no `revoke_key`. No HTTP routes for either. Once a user clicks Trust on the TOFU prompt by mistake, the key stays in the on-disk `known_hosts` store forever unless they edit the file by hand.
- **Fix:**
  1. Add `fn list_host_keys()` and `fn remove_key(hostname, port, key_type)` to `host_keys.rs`.
  2. Add `GET /api/host-keys` and `DELETE /api/host-keys/:hostname/:port` routes.
  3. Add a new "Trusted Host Keys" tab in Settings with list + per-row revoke button.

## P4 — CORRECTNESS / ASYNC HYGIENE

### P4-1. Async `useEffect` race conditions on 7 components — **XS each**
- Pattern: `useEffect(() => { const load = async () => { setState(await ...) }; load(); }, [deps])` with no mount/abort guard. Rapid dep changes let older fetches resolve after newer ones, clobbering state.
- Sites: `components/ProfilingAgentChat.tsx:43`, `AISettingsTab.tsx:379,540`, `AboutModal.tsx:22`, `workspace/WorkspaceHistoryEditor.tsx:28`, `mop/DeviceSelector.tsx:60`, `UpdateChecker.tsx:36`, `SftpFileBrowser.tsx:127`.
- **Fix pattern:**
  ```tsx
  useEffect(() => {
    const ac = new AbortController();
    (async () => {
      try {
        const data = await api(..., { signal: ac.signal });
        if (!ac.signal.aborted) setState(data);
      } catch (e) { if (!ac.signal.aborted) /* handle */ }
    })();
    return () => ac.abort();
  }, [deps]);
  ```

### P4-2. `useEffect` exhaustive-deps suppressions hide 7+ stale closures — **S total**
- 15 `// eslint-disable react-hooks/exhaustive-deps` sites. At least 7 hide real stale-dep bugs:
- `components/EnterpriseSessionPanel.tsx:180`, `AISidePanel.tsx:214`, `SharedTerminal.tsx:176`, `SftpPanel.tsx:304`, `TopologyTabEditor.tsx:653`, `hooks/useEnterpriseSSH.ts:408`, `hooks/useWorkspace.ts:242,334`, `Terminal.tsx:2654`.
- **Fix:** memoize the callback (wrap in `useCallback` with proper deps), then remove the suppression and add the callback to the effect deps. For state that needs to be "read-only at effect time", use a ref.

### P4-3. Polling without backoff or signal — **S total**
- `hooks/useWorkspace.ts:252` (1s `.netstacks/open-request.json` poll), `api/scripts.ts:297` (120s script poll), `EnterpriseSessionPanel.tsx:189` (15s, no 401 suppression), `hooks/useAIAgent.ts:2011` (3s, no AbortSignal).
- **Fix:** introduce a shared `usePoll(fn, { interval, backoff, signal })` hook. Exponential backoff on consecutive errors. AbortSignal to cancel on unmount.

### P4-4. AI JSON parse without shape validation — **S total**
- `hooks/useAIAgent.ts:1786-1790` (4 strings, one shared catch), `hooks/useNextStepSuggestions.ts:127`, `components/mop/MopWorkspace.tsx:1367,1533,1603,1668,1782`.
- **Fix:** use `parseAiResponse<T>(json, requiredFields)` pattern (already exists in `useAiPilot.ts:115-123` per the M12 fix). Add a `validateShape<T>` helper that takes a JSON-Schema-lite spec and returns `T | null`.

### P4-5. `Promise.all` where partial failure loses N-1 results — **XS each**
- `components/AISettingsTab.tsx:543` (one anthropic-vault error wipes openai/openrouter/custom), `SessionPanel.tsx:351`, `EnterpriseSessionPanel.tsx:158`, `AISidePanel.tsx:552`, `QuickActionsPanel.tsx:91`.
- **Fix:** switch to `Promise.allSettled`. Reduce to `{ value, error }` per leg, display per-leg state.

### P4-6. `eprintln!("[DEBUG] ...")` in production handlers — **XS**
- `agent/src/api.rs:2096,2128,2137,2167,5711,5771,5777,318`. Bypasses tracing, ignores `RUST_LOG`, leaks NetBox URLs / resource IDs to stderr.
- **Fix:** find/replace to `tracing::debug!` / `tracing::warn!`.

### P4-7. No database transactions anywhere — **M**
- `grep -rn 'begin()|begin_transaction|\.commit()|\.rollback()' agent/src` returns zero. Bulk operations execute as independent `sqlx::query(...).execute(pool)` calls. Mid-loop failure leaves the DB torn.
- **Fix:** wrap multi-statement flows in `let mut tx = pool.begin().await?; ... tx.commit().await?;` — start with the worst offenders: `api.rs:308` bulk delete sessions, `providers/local.rs` bulk credential operations, the MOP execution device-row writers.

### P4-8. Vault no `wait` on backend startup orphan eviction — **XS**
- `agent/src/main.rs:202-204` uses `std::net::TcpStream::connect_timeout(...)` which blocks the runtime thread for up to 200ms.
- **Fix:** replace with `tokio::time::timeout(Duration::from_millis(200), tokio::net::TcpStream::connect(addr))`.
